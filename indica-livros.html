Indica Livros

Desafio de Design Patterns desenvolvido no Bootcamp Santander, na plataforma de estudos DIO. API REST em Spring Boot para buscar livros na Google Books e registrar indicações com nota e motivo, com frontend web integrado, PostgreSQL versionado com Flyway e deploy em produção via Docker no Railway.

Spring Boot PostgreSQL Flyway Docker Design Patterns Railway

O que é este projeto

A aplicação permite buscar livros por título ou autor usando a Google Books API, selecionar um resultado e registrar uma indicação com nome, nota e motivo. As indicações ficam persistidas no PostgreSQL e aparecem na listagem da interface web.

  • GET /livros?titulo=... — busca por título
  • GET /livros?autor=... — busca por autor
  • GET /indicacoes — lista indicações salvas
  • POST /indicacoes — cadastra nova indicação

Stack e infraestrutura

  • Spring Boot 4 — API REST com controllers, services e JPA
  • PostgreSQL — persistência de livros e indicações
  • Flyway — migrations versionadas (livro, indicacao)
  • Docker — Postgres local via Compose; imagem multi-stage para produção
  • OpenFeign — cliente HTTP declarativo para a Google Books API
  • Railway — deploy da API e do banco em produção

Design Patterns aplicados

Strategy — busca por título ou autor

A interface BuscaLivroStrategy define o contrato de busca; implementações separadas (BuscaPorTituloStrategy e BuscaPorAutorStrategy) encapsulam a regra de cada tipo. O controller escolhe a strategy conforme o parâmetro da requisição, sem duplicar lógica.

Camadas e responsabilidades

Controllers expõem HTTP; services concentram regras de negócio; repositories acessam o banco via JPA; DTOs transportam dados entre camadas. A integração externa fica isolada em Feign clients com configuração própria para a API key da Google Books.
